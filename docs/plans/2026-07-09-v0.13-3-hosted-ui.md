# v0.13 (3/4) — Hosted Board UI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fourth board mode `"hosted"` to the React board: collaborators comment on a password-gated hosted board, each Save POSTs to the comments API, the board GETs all comments on load with shared visibility, per-document staleness demotes only comments on changed documents, a failed Save never loses text, and a "Publish to web" button appears in live mode. All comment logic lives in a pure, lib-tested module; the App wiring is thin and verified by build + the sub-plan 4 e2e.

**Architecture:** One new pure module `board/src/lib/hostedComments.ts` (uuid/clientId, per-document staleness, two-population state, failed-post handling) tested at the lib level, plus targeted edits to `board/src/lib/types.ts`, `board/src/lib/feedback.ts`, and `board/src/App.tsx`, then one rebuild of the single-file template. Consumes the API contract frozen in sub-plan 2 (`POST`/`GET /api/comments`). The publish button POSTs to the local server's `/publish-web` (added in sub-plan 4); until then it degrades to a "run the command" hint.

**Tech Stack:** Vite + React + TypeScript, node-environment `vitest` (no jsdom — lib tests only, matching the repo), `vite-plugin-singlefile` build → `skills/managing-research-plans/assets/board-template.html`.

**Spec:** `docs/specs/2026-07-09-updates-sharing-models-design.md` — v0.13 §4 (hosted mode in the UI) and the Testing harness note. Read before starting.

## Global Constraints

- **Node-env vitest only.** `board/` has no jsdom/@testing-library. Put ALL testable logic in `hostedComments.ts` and test it at the lib level; App.tsx rendering is not unit-tested (verified by `npm run build` here and the real-Vercel e2e in sub-plan 4).
- **Two comment populations, separate state.** *Server comments* (from GET) render read-only — no delete button. *Local pending comments* (not yet saved) are editable/deletable. Server comments are NEVER written into the annotations localStorage array (else they duplicate on reload); only pending items are persisted; only a posted item is removed from pending.
- **Real UUID ids** for hosted comments (`crypto.randomUUID`), not `ann-<ts>-<counter>` — the API upsert idempotency depends on a stable client id.
- **A failed Save keeps the text.** On network error or `401`, the comment stays pending and a banner appears ("Couldn't save — re-enter the password; your unsent comments are kept").
- **Pending drafts + name survive republish.** Keyed by **project + board URL**, NOT `payloadHash` (which changes every republish and would orphan drafts / reset the name).
- **`author` + `clientId` on every posted comment.** Name field required before first save ("Full name — visible to all collaborators"). `clientId` is a random per-browser id so `--pull` can split same-named collaborators.
- **Per-document staleness.** A comment is demoted to a marked "written before the board was last updated" section only when ITS target document changed; the copy reassures "the researcher still has a copy of all comments." Never paint stale comments as current.
- **Immutable microcopy + post-save confirmation.** "Comments can't be edited or deleted once sent" at the save affordance; "Sent — visible to everyone with this link; the researcher picks comments up in Claude Code" after Save.
- **Mobile.** Board reads well on a phone; a banner on touch/narrow viewports: "Reading works here; commenting works best on a computer." (Touch commenting is future work — the gesture is `onMouseUp`-only at `AnnotationLayer.tsx:121`.)
- **Commit messages:** plain, no `Co-Authored-By`. Do not push.

```ts
import type { Annotation, BoardData, StoredComment } from "./types";

export function newUuid(): string {
  if (typeof crypto !== "undefined" && "randomUUID" in crypto) return crypto.randomUUID();
  let hex = "";
  for (let i = 0; i < 32; i++) hex += Math.floor(Math.random() * 16).toString(16);
  return `${hex.slice(0, 8)}-${hex.slice(8, 12)}-4${hex.slice(13, 16)}-8${hex.slice(17, 20)}-${hex.slice(20, 32)}`;
}

const CLIENT_KEY = "rp-board:clientId";
export function getClientId(storage: Storage): string {
  let id = storage.getItem(CLIENT_KEY);
  if (!id) { id = newUuid(); storage.setItem(CLIENT_KEY, id); }
  return id;
}

function hashContent(s: string): string {
  let h = 0x811c9dc5;
  for (let i = 0; i < s.length; i++) { h ^= s.charCodeAt(i); h = Math.imul(h, 0x01000193); }
  return (h >>> 0).toString(16).padStart(8, "0");
}

/** The content of the annotation's target document, if file-backed; else null. */
export function targetHash(data: BoardData, a: Annotation): string | null {
  const exec = (data as { exec?: Array<Record<string, unknown>> }).exec ?? [];
  const findComp = (c: string) => exec.find((g) => g.component === c);
  if (a.type === "plan-comment") {
    const g = findComp(a.component) as { versions?: Array<{ version: number; content: string }> } | undefined;
    const v = g?.versions?.find((x) => x.version === a.version);
    return v ? hashContent(v.content) : null;
  }
  if (a.type === "result-comment" || a.type === "script-comment") {
    // File-backed under the component's results; hash the component's results blob.
    const g = findComp(a.component) as { results?: unknown } | undefined;
    return g?.results ? hashContent(JSON.stringify(g.results)) : null;
  }
  // doc-comment (a derived view) and general: no single file — fall back to board hash.
  return null;
}

export function isStale(c: StoredComment, data: BoardData): boolean {
  if (c.docHash != null) return targetHash(data, c.annotation) !== c.docHash;
  return c.shareHash !== data.shareHash;
}

export function partitionComments(
  server: StoredComment[], data: BoardData,
): { live: StoredComment[]; stale: StoredComment[] } {
  const live: StoredComment[] = []; const stale: StoredComment[] = [];
  for (const c of server) (isStale(c, data) ? stale : live).push(c);
  return { live, stale };
}

export function applyPostResult(pending: Annotation[], id: string, ok: boolean): Annotation[] {
  return ok ? pending.filter((a) => a.id !== id) : pending;
}

export function buildCommentBody(a: Annotation, data: BoardData, author: string, clientId: string) {
  return {
    id: a.id && /^[0-9a-f-]{36}$/i.test(a.id) ? a.id : newUuid(),
    clientId, author, shareHash: (data as { shareHash?: string }).shareHash ?? "",
    docHash: targetHash(data, a),
    annotation: a,
  };
}
```

  async function saveHosted(a: Annotation) {

    try {
      const res = await fetch("/api/comments", {
        method: "POST", headers: { "content-type": "application/json" },

        shareHash: data.shareHash ?? "", docHash: null, annotation: a, receivedAt: "",
      }]);

      setSaveError(true); // comment stays in `annotations` (pending) — not lost

  }
